# ILS — Go-to-Market Action List

## Phase 1 — Product Identity

1. Choose the product name
2. Check domain availability and buy the domain
3. Define the one-line positioning statement ("ILS is the platform that...")
4. Define your ideal customer (MBO / HBO / who signs the contract)
5. Create brand identity — logo, colors, tagline

---

## Phase 2 — Marketing Website

6. Build the public marketing website (separate from the app)
7. Design the Home page: Hero → Problem → Solution → Features → Screenshots → CTA
8. Write a feature page for each major area:
   - Teacher Workspace
   - Student Success
   - Curriculum Management
   - Programme Monitoring
   - Learning Analytics
   - Assessment & Rubrics
   - Attendance
   - Internships
   - Data Exchange
   - Reflection & Feedback
9. Write the About page — your story, mission, why you built ILS
10. Write the **Why ILS?** page — philosophy vs traditional LMS
11. Write the Pricing page — decide model first (per institution / per student / per user)
12. Write the Security & GDPR page
13. Add Contact / Request Demo flow

---

## Phase 3 — Trust & Credibility

14. Write public Documentation:
    - Getting Started
    - Teacher Guide
    - Coordinator Guide
    - Student Guide
    - Administrator Guide
15. Write FAQ (real questions schools will ask — GDPR, import, SSO, backups, implementation time)
16. Publish a public Roadmap

---

## Phase 4 — Demonstration

17. Build a demo environment with realistic data (multiple programmes, teachers, students)
18. Create demo accounts: Teacher / Coordinator / Student / Admin
19. Produce short demo videos:
    - 2 min — Teacher workflow
    - 2 min — Coordinator workflow
    - 90 sec — Student experience
    - 3 min — Curriculum designer
    - 3 min — Administrator
20. Take professional screenshots for the website

---

## Phase 5 — Commercial Readiness

21. Define the sales process — what happens after someone requests a demo?
22. Design the onboarding flow — institution created, admin invited, curriculum imported, go live
23. Define the migration/import story — Canvas, Moodle, Excel, CSV
24. Prepare training materials — short videos + PDF guides per role

---

## Phase 6 — Launch

25. Create LinkedIn company page
26. Start sharing development journey and educational insights (not ads — build authority)
27. Launch a newsletter / waitlist to capture early interest
28. Pilot with 1–2 institutions — gather feedback and testimonials
29. Write first case study
30. Announce publicly

---

## Suggested Priority Order

If starting today, work in this order:

1. Finalize the product name and secure the domain
2. Create brand identity (logo, colors, tagline)
3. Build the public marketing website
4. Produce screenshots and demo videos
5. Prepare documentation and onboarding materials
6. Define pricing and the sales/demo process
7. Launch a public demo environment
8. Begin sharing on LinkedIn and through education networks
9. Pilot with 1–2 institutions
10. Announce publicly

---

## Key Positioning Insight

> ILS is not another LMS.
>
> ILS connects curriculum design, teaching, assessment, student support and institutional intelligence into one platform.
>
> Canvas doesn't do that. Moodle doesn't do that.

**That is the story. Lead with it everywhere.**

## Who Buys vs Who Uses

| Role | Relationship |
|---|---|
| Programme Manager | Buyer |
| Education Manager | Buyer |
| Department Head | Buyer |
| School Director | Buyer |
| Teacher | User |
| Coordinator | User |
| Student | User |

The website must speak to both — buyers need the business case, users need to see it works.

## Why ILS? (comparison table for website)

| Traditional LMS | ILS |
|---|---|
| Focus on content delivery | Focus on learning outcomes |
| Separate disconnected systems | Integrated end-to-end platform |
| Reports after the fact | Operational intelligence in real time |
| Teacher-centered | Institution-wide insights |
