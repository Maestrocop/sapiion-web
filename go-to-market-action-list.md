# ILS — Go-to-Market Action List
> Version 3 — refined with strategic anchors

---

## The ILS Product Principle

> **Every feature in ILS should contribute to improving educational quality, not just digitizing educational administration.**

This is the filter for every product and marketing decision.

---

## Product Category

> **ILS — Intelligent Learning System**
> An Educational Intelligence Platform for continuous improvement.

ILS is not a next-generation LMS (too small — invites Canvas/Moodle comparison).
ILS is not a Learning Analytics Platform (too narrow — becomes a reporting tool).
ILS is a **Learning Operations Platform** — the operating system for modern education.

---

## Core Positioning

> ILS is an educational intelligence platform that connects curriculum design, teaching, assessment and student success into one continuous improvement cycle.

## Phase 1 — Strategy (Foundation)

Do not skip. Everything else is built on this.

1. Write **PRODUCT_VISION.md** (internal)
   - Why does ILS exist?
   - What is the long-term vision?
   - What educational philosophy guides ILS?

2. Write **EDUCATIONAL_MODEL.md** (internal + future public)
   - Define the ILS educational philosophy

     Curriculum Layer → Teaching Layer → Assessment Layer → Student Success Layer → Institution Intelligence Layer

## Phase 5 — Launch

31. Create LinkedIn company page, start sharing educational insights (not ads — build authority)
32. Launch newsletter / waitlist to capture early interest
33. Run **Early Adopter Programme** (structured pilot):
    - Objectives: validate workflows, measure implementation effort, collect testimonials, discover missing features
    - Deliverables: feedback reports, case studies, feature requests, success metrics
34. Form **Customer Advisory Group** after pilots:
    - Members: programme managers, teachers, educational advisors
    - Purpose: validate roadmap, test ideas, prevent building in isolation
35. Write first case study from pilot
36. Announce publicly

- Time to first class created

### Commercial
- Demo → Pilot conversion rate
- Pilot → Customer conversion rate
- Customer retention rate

## Product Roadmap Governance

After launch, define how work gets prioritized:

- **P0** — Critical bugs (fix immediately)
- **P1** — Customer-blocking issues (next release)
- **P2** — Feature requests from pilots (quarterly review)

## Why ILS? (for the website)

| Traditional LMS | ILS |
|---|---|
| Focus on content delivery | Focus on learning outcomes |
| Separate disconnected systems | Integrated end-to-end platform |
| Reports after the fact | Operational intelligence in real time |
| Teacher-centered | Institution-wide insights |
| Content → Assignment → Grade | Intent → Design → Evidence → Mastery → Improvement |
